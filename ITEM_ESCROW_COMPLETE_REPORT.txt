BidMarket Item Escrow Complete Update

ฐานที่ใช้: BidMarket-item-escrow-update.zip

สิ่งที่ทำต่อให้สมบูรณ์:
1. หน้าเมนูรูปค้อน
   - ลบ/ลดความซ้ำของหัวข้อ ซื้อ/ขาย และใช้หัวข้อ กลางไอเทม เป็นเมนูหลักสำหรับระบบนี้
   - คงหน้า market เป็นหน้ากลางไอเทม

2. ฟอร์มลงขายกลางไอเทม
   - ผู้ขายต้องผ่าน KYC ก่อนลงขาย
   - บังคับกรอกชื่อไอเทม รายละเอียด ราคา ชื่อตัวละครผู้ขาย
   - รองรับอัปโหลดรูปสินค้าจริงหลายรูปผ่าน multipart/form-data
   - ยังรองรับ URL รูปเป็นช่องสำรอง
   - มีตัวเลือก ผู้ขายจ่ายค่าธรรมเนียมให้ผู้ซื้อ
   - เพิ่ม preview ค่าธรรมเนียม/ยอดผู้ซื้อจ่าย/ยอดผู้ขายรับสุทธิใน UI

3. ฟอร์มซื้อกลางไอเทม
   - ผู้ซื้อกรอกชื่อตัวละครผู้ซื้อก่อนชำระเงิน
   - ระบบล็อก Credit ตามราคาสินค้า + ค่าธรรมเนียม หรือราคาสินค้าอย่างเดียวหากผู้ขายรับค่าธรรมเนียม

4. ค่าธรรมเนียม
   - <= 100 Credit คิด 7%
   - >100 ถึง <=500 Credit คิด 10 Credit
   - >500 Credit คิด 5%
   - เศษทศนิยมมากกว่า 0.3 ปัดขึ้น, เท่ากับ/น้อยกว่า 0.3 ปัดลง

5. ส่งมอบ + Admin Check
   - ผู้ขายต้องแนบรูปหลักฐานส่งมอบอย่างน้อย 1 รูป
   - ผู้ขายระบุตัวละครที่ส่งมอบจริง + หมายเหตุ
   - หลังส่งมอบ สถานะเป็น PENDING_ADMIN_CHECK
   - แจ้งเตือน Admin ผ่านระบบแจ้งเตือนในเว็บ
   - Admin ดูรายละเอียดคำสั่งซื้อ/หลักฐาน/ตัวละครผู้ซื้อ/ตัวละครที่ส่งจริง
   - Admin กด อนุมัติ หรือ ตีกลับให้ผู้ขายส่งหลักฐานใหม่
   - เมื่อ Admin อนุมัติจึงแจ้งผู้ซื้อให้กดยืนยันรับสินค้า

6. หน้า Orders
   - แสดงรายละเอียดกลางไอเทมครบ: ราคา, ค่าธรรมเนียม, Credit ที่ล็อก, ตัวละครผู้ขาย/ผู้ซื้อ/ส่งมอบจริง, รูปหลักฐาน
   - ผู้ขายอัปโหลดหลักฐานจากหน้า Orders ได้
   - ผู้ซื้อกดยืนยันรับสินค้าได้เฉพาะหลัง Admin อนุมัติ
   - มีปุ่มแจ้งปัญหา/ข้อพิพาทพร้อมแนบไฟล์

7. หน้า Admin
   - เพิ่มคิว Admin Check กลางไอเทมด้านบน Admin Dashboard
   - ปุ่มดูรายละเอียดกลางไอเทม
   - ปุ่มอนุมัติหลักฐานส่งมอบ
   - ปุ่มตีกลับหลักฐานให้ผู้ขายส่งใหม่
   - แสดงรูปหลักฐานส่งมอบและหลักฐานข้อพิพาท

8. ระบบเงิน
   - แก้คืนเงินให้คืนตาม locked_amount เพื่อให้กรณีผู้ซื้อจ่ายค่าธรรมเนียมได้รับคืนครบ
   - เมื่อปล่อยเงินบันทึกรายได้บริษัทจากค่าธรรมเนียมกลางไอเทม
   - บันทึก audit / escrow event ทุกขั้นตอนสำคัญ

ตรวจสอบ:
- node --check server.js ผ่าน
- node --check public_inline.js ผ่าน
- ตรวจ parse JS patch เพิ่มเติมผ่าน
